# Skills Data Structure

This directory contains individual skill JSON files for better organization and maintainability.

## Structure

```
skills/
├── index.json                          # Lightweight index of all skills
├── {skill-name}.json                   # Individual skill files
├── calculator.json
├── example.json
├── pet-health-advice.json
├── reactjs-best-practices.json
├── reactjs-component-generator.json
├── text-processor.json
├── vercel-deploy.json
└── web-design-guidelines.json
```

## File Contents

### Individual Skill Files

Each `{skill-name}.json` file contains the complete skill data:

```json
{
  "metadata": {
    "name": "skill-name",
    "description": "Skill description...",
    "category": "category-name",
    "tags": ["tag1", "tag2"],
    "version": "1.0.0",
    "owner": "owner-name"
  },
  "content": "# Skill Content\n\nFull markdown content...",
  "path": "/path/to/skill",
  "installCount": 1234,
  "trendingScore": 85,
  "createdAt": "2025-01-01T00:00:00.000Z",
  "updatedAt": "2025-01-18T00:00:00.000Z"
}
```

### Index File

The `index.json` file contains lightweight metadata for all skills:

```json
[
  {
    "name": "skill-name",
    "description": "Skill description...",
    "category": "category-name",
    "tags": ["tag1", "tag2"],
    "installCount": 1234,
    "trendingScore": 85,
    "createdAt": "2025-01-01T00:00:00.000Z",
    "updatedAt": "2025-01-18T00:00:00.000Z"
  }
]
```

## Usage

### Loading All Skills

```typescript
import { getAllSkills } from '@/lib/skills.server'

const skills = getAllSkills()
```

### Loading Skills Index (Lightweight)

```typescript
import { getSkillsIndex } from '@/lib/skills.server'

// Only loads metadata, not full content
const skillsIndex = getSkillsIndex()
```

### Loading Individual Skill

```typescript
import { loadSkillByName } from '@/lib/skills.server'

const skill = await loadSkillByName('calculator')
```

### Direct Import

```typescript
import calculatorSkill from '@/data/skills/calculator.json'
import skillsIndex from '@/data/skills/index.json'
```

## Benefits

1. **Better Organization**: Each skill in its own file makes it easier to find and edit
2. **Version Control**: Smaller, focused diffs when skills are updated
3. **Lazy Loading**: Load only the skills you need
4. **Maintainability**: Easier to add, remove, or update individual skills
5. **Reduced Bundle Size**: Import only what you need

## Generation

Skills are generated by the `scripts/generate-web-data.ts` script:

```bash
npm run generate:data
```

This script:
1. Reads skills from `@repo/skills-kit`
2. Generates individual skill files
3. Creates the index file
4. Updates search mappings

## Backward Compatibility

The parent directory still contains `skills.json` for backward compatibility:

```typescript
import skills from '@/data/skills.json' // Still works
```

However, new code should prefer the individual skill files approach.
